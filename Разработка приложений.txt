1. Постановка задачи.
Разработать игру "Морской бой".

2. Описание игры.
2 поля 10х10, одно свое, второе соперника.
В своем поле устанавливаются корабли: 
4 клетки - 1 шт; 
3 клетки - 2 шт.; 
2 клетки - 3 шт.;
1 клетка - 4 шт. 
Корабли могут распологаться горизонтально и вертикально. Расстояние между 
кораблями не менее 1 клетки. Корабли не должны пересекаться друг с другом. 
Предусмотреть возможность расстановки своих кораблей компьютером.
Ходы делаются по очереди. На поле противника указывается точка выстрела.
Варианты ответа: "мимо", "ранил", "убил". При удачном выстреле ход не
передается. При ответе "убил" окружаем уничтоженный корабль
недоступными для выстрела клетками. Игра продолжается до уничтожения всех кораблей
одного из играющих.

3. Разработка 

3.1 Viewer (оболочка)
На форме 2 поля 10х10 клеток. Ниже полей показаны корабли. Свои корабли перед 
началом игры выставляем на игровое поле. Корабли противника подсвечиваются
по мере уничтожения.
Над полями надпись о принадлежности хода.
Клетки поля противника могут быть в 3х состояниях:
1. свободное поле, доступное для выстрела;
2. поле попадения;
3. неактивное, окружающее уничноженный корабль.
Правее находятся кнопки - "Начать игру" и "Сдаться","Расставить корабли автоматически".

3.2 Модель (model).
Перечисление статуса выстрела - Мимо, Ранил, Убил, Конец боя.
Перечисление статуса координат - Пусто, Корабль, Выстрел, Попал.
Перечисление типы кобалей - 4х,3х,2х,1х.
Перечесление направление размещения корабля: горизонтально, вертикально.
Поле в котором храняться координаты своих кораблей - двумерный массив, элементами
которого являются: 0 - пустое поле, 1- есть корабль или его часть, 2 - выстрел, 
3 - попавший выстрел. 
Поле в котором храняться координаты кораблей противника - двумерный массив, элементами
которого являются: 0 - пустое поле, 1- есть корабль или его часть, 2 - выстрел, 
3 - попавший выстрел. 
Поле количества неубитых частей кораблей. При инициализации - 20.
Поле статуса последнего выстрела.
Поле координат последнего выстрела (строка из 2х цифр).
Поле ранения логическое.

Метод автоматического заполнения координаты корабля.
Метод автоматического заполнения координат кораблей.
Метод заполнения координат кораблей вручную.
Метод отчистки координат кораблей.
Метод удаления координат корабля.
Метод выстрела компьютера(генерация случайных координат).
Метод выстрела игрока.
Метод проверки правильности координат корабля.

3.2.1 Метод выстрела игрока.
Входящие параметры - координаты выстрела (x,y) - строка из 2х цифр.
Результат - перечисление статуса выстрела.
При пустом поле координат (Пусто) возвращаем "Мимо".
При не пустом поле координат проверяются точки с координатами x+1,x-1,y+1,y-1 для 
анализа статуса корабля. Если в этих координатах есть "корабль", то возвращаем "Ранил", 
иначе - "Убил". Меняется статус координат. Уменьшаем счетчик неубитых частей кораблей.
Если он стал равен 0, то возвращаем Конец боя.

3.2.2 Метод выстрела компьютера (генерация случайных координат)
Входящих параметров нет. Возвращает строку из двух цифр -
координат массива кораблей противника. 
Проверяем статус полседнего выстрела. Если статус "Убил", то поле ранения ложь. Если статус "Мимо" 
или "Убил" и поле ранение ложь, то генерируем случайное число.
Если "Ранил",то поле ранения истина, то в массиве координат противника проверяем координаты x-1, x+1, y-1, y+1.
Если есть выстрелы по этим координатам, то генерируем координаты в противоположную сторону.
Если выстрелов не было (первое попадание), то случайным образом выбираем одно из проверяемых 
значений.
Если статус "Мимо" и поле ранения истина, то проверяем координаты x-2, y-2, x+2, y+2.
При генерации проверяем полученные координаты, если в массиве элемент с этими координатами
не пустое поле, то генерируем новые координаты.
 
3.2.3 Метод заполнения координат кораблей вручную.
Принимает координаты начала корабля, тип корабля, направление размещения.
Направление размещения по умолчанию вертикально.
Заполняет массив координат кораблей.

3.2.4 Метод удаления координат корабля.
Принимает координаты начала корабля, тип корабля, направление размещения.
Направление размещения по умолчанию вертикально.
Удаляет из массива координат корабль.

3.2.5 Метод отчистки координат кораблей.
Входных параметром нет. Очищает массив координат кораблей.

3.2.6 Метод автоматического заполнения координаты корабля.
Входные параметры: Тип корабля и направление.
Случайным образом получает координаты начала корабля, вызывает метод проверки правильности координат корабля,
если координатя соответствуют, то помещаем в поле координат своих кораблей.

3.2.7 Метод автоматического заполнения координат кораблей.
Не принимает параметров.
Вызывает метод автоматического заполнения координаты корабля для всех кораблей.
Парамет направление берется случайным образом.
 
3.2.8 Метод проверки правильности координат корабля.
Входные парметры: тип корабля, координатя корабля, направление корабля.
Результат логический.
Проверяем соответствие каждой клетки корабля правилам, если не соответствует - то возвращаем ложь.
